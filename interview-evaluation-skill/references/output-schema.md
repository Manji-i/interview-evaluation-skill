# 输出结构

生成候选人面试评价时使用以下结构。没有证据的字段不要硬编，标注“未提供”或“证据不足”。

```markdown
## 结论先行

- 总体建议：强推进 / 推进 / 不推进
- 总体评级：NH / H- / H / H+ / MH
- 核心评价：一句话总结。不仅评价能力，还要评价维度匹配度。必须来自证据。
- 主要风险：1-3 条，必须来自证据。

## 候选人基础概览

- 岗位/职级：[当前岗位/职级]
- 看机会原因：[简述原因]
- 面试/Offer 情况：[公司名及进展]
- 过往绩效：[如：连续 S / 前 10% 等]
- 核心工作：[一句话总结：负责...实现...]

## 业务架构与组织环境

- 业务背景：[描述其所在业务的逻辑、规模或技术栈]
- 汇报/组织关系：[汇报给谁，与哪些部门平行，在组织中的位置，部门多少人]

## 指定维度详细评估

### [用户指定的维度名称 1]

- 评分：NH / H- / H / H+ / MH
- 证据：
- 匹配度分析：[基于维度要求的评价。例如：对于【学习能力】，此案例展现的学习难度较低，学习目标挑战小，学习方法常规，好奇心不足，评分为 NH。]

### [用户指定的维度名称 2]

- 评分：NH / H- / H / H+ / MH
- 证据：
- 匹配度分析：

## 额外观察

- 额外加分项：[如发现用户未列出但对岗位至关重要的正向能力，写明证据；没有则写“无”。]
- 额外减分项：[如发现用户未列出但对岗位至关重要的风险，写明证据；没有则写“无”。]
```

维度评估部分按用户指定维度循环输出。不要在“指定维度详细评估”中额外添加用户未确认的维度；如有额外发现，统一放入“额外观察”。如果某个维度证据不足，仍保留该维度并说明证据不足。
